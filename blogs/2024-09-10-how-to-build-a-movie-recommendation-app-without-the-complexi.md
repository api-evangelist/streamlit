<figure><img alt="" src="https://cdn-images-1.medium.com/max/1024/1*tGu9BKtPZqTReeBH6__DsA.png" /></figure><h4>Use the Streamlit-Weaviate Connection to integrate a vector database</h4><p><em>Originally published on the </em><a href="https://blog.streamlit.io/how-to-recommendation-app-vector-database-weaviate/"><em>Streamlit blog</em></a><em> by </em><a href="https://blog.streamlit.io/author/liz/"><em>Liz Acosta</em></a></p><p>You are what you eat; your model is what your model ingests.</p><p>Not only does data inform AI systems, data is the output you ultimately receive. That’s why it’s important to have “good” data. It doesn’t matter how powerful your model is, garbage in will always result in garbage out.</p><p>In software development, this isn’t a new concept or problem. However, AI demands a more sophisticated data strategy throughout the ETL process. This can slow the delivery of your AI-integrated applications.</p><p>In this recipe, you’ll use Weaviate to abstract away the complexity associated with vector databases, allowing you to implement a powerful search and recommendation system with way less technical overhead. Then we’ll use Streamlit to build the chatbot part of the app.</p><p>And don’t panic! There’s no frontend involved!</p><p>Read on to learn:</p><ul><li>What Weaviate is</li><li>What Streamlit is</li><li>How to build a demo Weaviate movie recommendation Streamlit app</li><li>How to query a Collection in Weaviate Cloud</li></ul><p>Don’t feel like reading? Here are some other ways to explore this demo:</p><ul><li><a href="https://github.com/streamlit/cookbook/tree/main/recipes/weaviate">Find the code in the Streamlit Cookbook repo</a></li><li><a href="https://www.youtube.com/watch?v=SQD-aWlhqvM">Watch a video walkthrough</a> with Weaviate technical curriculum developer, JP Hwang</li><li><a href="https://weaviate-movie-magic.streamlit.app/">Check out a deployed version of the app</a></li></ul><h3>What is Weaviate?</h3><p><a href="https://weaviate.io/">Weaviate</a> is an AI-native database designed to help you build amazing, scalable, and production-grade AI-powered applications. It offers robust features for data storage, retrieval, and querying as well as integrations with AI models, making it an excellent choice for developers looking to integrate AI capabilities into their apps.</p><h3>The Streamlit-Weaviate Connection</h3><p>The <a href="https://github.com/weaviate/st-weaviate-connection">Streamlit-Weaviate connection</a> is a wrapper that simplifies the process of integrating Weaviate with Streamlit applications. This connection allows you to perform various operations, such as connecting to a remote or local Weaviate instance, performing queries, and using the underlying Weaviate Python client. The project is open-source so contributions are always welcome.</p><p><strong>Key features</strong></p><ul><li><strong>Connect to a Weaviate Cloud instance:</strong> Easily connect to a Weaviate cloud instance using a URL and API key</li><li><strong>Connect to a locally running Weaviate instance:</strong> Easily connect to a Weaviate instance running locally</li><li><strong>Perform queries:</strong> Execute simple and advanced queries using the query or GraphQL query methods</li><li><strong>Use the Weaviate Python client:</strong> Leverage the full capabilities of the Weaviate Python client for more complex operations</li><li><strong>Support for local instances:</strong> Connect to a local Weaviate instance using default parameters</li><li><strong>Secrets management:</strong> Streamlit can handle secret management for secure connections</li></ul><h3>What is Streamlit?</h3><p>Streamlit is an open-source Python framework to build highly interactive apps — in only a few lines of code. Streamlit integrates with all the latest tools in <a href="https://streamlit.io/generative-ai">generative AI</a>, such as any LLM, vector database, or various AI frameworks like <a href="https://blog.streamlit.io/langchain-streamlit/">LangChain</a>, <a href="https://blog.streamlit.io/build-a-chatbot-with-custom-data-sources-powered-by-llamaindex/">LlamaIndex</a>, or Weights &amp; Biases. Streamlit’s <a href="https://docs.streamlit.io/develop/api-reference/chat">chat elements</a> make it especially easy to interact with AI so you can build chatbots that “talk to your data.”</p><p>Combined with a platform like Replicate, Streamlit allows you to create generative AI applications without any of the app design overhead.</p><p>To learn more about Streamlit, <a href="https://blog.streamlit.io/streamlit-101-python-data-app/">check out the 101 guide</a>.</p><p>💡 To learn more about how Streamlit biases you toward forward progress, check out this <a href="https://blog.streamlit.io/just-build-it-streamlit-opinionated-framework/">blog post</a>.</p><h3>Try the app recipe: Weaviate + Streamlit</h3><p>In this demo, you’ll spin up a movie recommendation app that utilizes Weaviate for backend data management and Streamlit chat elements on the frontend for interaction. The app accepts a natural language input from a user and uses Weaviate to translate the input into a query and then generate a list of movie titles.</p><p>There are three different kinds of search modes available:</p><p><strong>Keyword:</strong> This search mode uses BM25 to rank documents based on the relative frequencies of search terms. In this particular app, that means the results returned are based on how often the search keywords appear in the different movie properties.</p><p><strong>Semantic:</strong> This type of search uses vectors to generate results based on their similarity to your search query. In other words, the results returned are based on a similarity of “meaning.” To learn more about vector databases, check out Weaviate’s <a href="https://weaviate.io/blog/what-is-a-vector-database"><em>Gentle Introduction to Vector Databases</em></a>.</p><p><strong>Hybrid:</strong> A hybrid search combines vector and BM25 searches to offer best-of-both-worlds search results.</p><h3>Prerequisites</h3><ul><li>Python &gt;=3.8, !=3.9.7</li><li><a href="http://console.weaviate.cloud">A Weaviate sandbox instance<br /></a>(This will give you the API and URL to use in the code as well)</li><li><a href="https://dashboard.cohere.com/welcome/register">A Cohere API key</a></li></ul><p>In this app, the Cohere API is used for two different operations:</p><ul><li>To summarize the query results into a natural language recommendation</li><li><a href="https://weaviate.io/developers/weaviate/model-providers/cohere">To transform the MovieDemo data and queries into high-dimensional vector representations</a> for the semantic and hybrid search modes</li></ul><p>Please note that both <a href="https://weaviate.io/pricing">Weaviate</a> and <a href="https://docs.cohere.com/docs/rate-limits">Cohere</a> have limits on their trial accounts. Check their websites for more details.</p><p>💡 To learn more about API keys, check out the blog post <a href="https://blog.streamlit.io/8-tips-for-securely-using-api-keys/">here</a>.</p><h3>Environment setup</h3><p><strong>Local setup: Create a virtual environment</strong></p><ol><li>Clone the Cookbook repo: git clone <a href="https://github.com/streamlit/cookbook.git`">https://github.com/streamlit/cookbook.git</a></li></ol><p>2. From the Cookbook root directory, change directory into the recipe: cd recipes/weaviate</p><p>3. Add the necessary secrets to the .streamlit/secrets_template.toml file:</p><pre>WEAVIATE_API_KEY = &quot;your weaviate key goes here&quot;<br />WEAVIATE_URL = &quot;your weaviate url goes here&quot;<br />COHERE_API_KEY = &quot;your cohere api key goes here&quot;</pre><p>4. Update the filename from secrets_template.toml to secrets.toml: mv .streamlit/secrets_template.toml .streamlit/secrets.toml</p><p>(To learn more about secrets handling in Streamlit, refer to the documentation <a href="https://docs.streamlit.io/develop/concepts/connections/secrets-management">here</a>.)</p><p>5. Create a virtual environment: python3 -m venv weaviatevenv</p><p>6. Activate the virtual environment: source weaviatevenv/bin/activate</p><p>7. Install the dependencies: pip install -r requirements.txt</p><p><strong>Add data to your Weaviate Cloud</strong></p><ol><li>Create a Weaviate Cloud <a href="https://weaviate.io/developers/weaviate/config-refs/schema#introduction">Collection</a> and add data to it: python3 helpers/add_data.py</li><li>(Optional) Verify the data: python3 helpers/verify_data.py</li></ol><h3>Query the MovieDemo collection in Weaviate Cloud</h3><p>You can access the <a href="https://weaviate.io/developers/weaviate/connections/connect-query#example-query">Query</a> panel via the Weaviate Cloud UI.</p><ol><li>Copy and paste the following query in the editor:</li></ol><pre>{ Get <br /> {MovieDemo (limit: 3 where: {<br /> path: [&quot;release_year&quot;],<br /> operator: Equal,<br /> valueInt: 1985}){<br /> budget<br /> movie_id<br /> overview<br /> release_year<br /> revenue<br /> tagline<br /> title<br /> vote_average<br />}}}</pre><p>2. Click on the arrow to execute the query</p><figure><img alt="" src="https://cdn-images-1.medium.com/max/1024/0*ON9MxhYWZ-YgtpoA" /></figure><p>The Weaviate Cloud Query tool is a browser-based GraphQL IDE. In the example query above, we are telling Weaviate to return the budget, movie_id, overview, release_year, revenue, tagline, title, vote_average properties for the objects in the MovieDemo collection with a release_year of 1985. We do this by setting the path to [&quot;release_year&quot;], the operator to Equal, and the valueInt to 1985. We also limit the query results to three objects with limit: 3.</p><p>You should get back a result of three movies with the release year 1985.</p><p>This is a simple query that forms the foundation of more complex queries. To learn more about different kinds of searches available with the Weaviate Cloud Query tool, <a href="https://weaviate.io/developers/weaviate/search/basics">check out the documentation</a>.</p><p>Run the demo Weaviate Streamlit recommendation app</p><p>To run the demo app, use the Streamlit CLI: streamlit run demo_app.py.</p><p>Running this command deploys the app to a port on localhost. When you access this location, you should see a Streamlit app running. Please note that this version of the demo app does not feature the poster images so it will look different from the <a href="https://weaviate-movie-magic.streamlit.app/">deployed app</a>.</p><figure><img alt="" src="https://cdn-images-1.medium.com/max/752/1*9yb6cVOa6hefTBsphZbvoQ.gif" /></figure><h3>Vector databases made easy</h3><p>Using Streamlit-Weaviate Connection means you can easily create and integrate vector databases in your Streamlit apps.</p><p>In demo_app.py, the Weaviate connection is created here:</p><pre>def setup_weaviate_connection(env_vars):<br />&quot;&quot;&quot;Setup Weaviate connection&quot;&quot;&quot;<br />  return st.connection(<br />    &quot;weaviate&quot;,<br />    type=WeaviateConnection,<br />    url=env_vars[&quot;WEAVIATE_URL&quot;],<br />    api_key=env_vars[&quot;WEAVIATE_API_KEY&quot;],<br />    additional_headers={&quot;X-Cohere-Api-Key&quot;: env_vars[&quot;COHERE_API_KEY&quot;]},)</pre><p>Using Streamlit <a href="https://docs.streamlit.io/develop/api-reference/chat">chat elements</a>, a prompt is created and a query is made here:</p><pre>with conn.client() as client:<br />collection = client.collections.get(&quot;MovieDemo&quot;)<br />response = collection.generate.hybrid(<br />query=movie_type,<br />filters=(<br />Filter.by_property(&quot;release_year&quot;).greater_or_equal(year_range[0]) &amp;<br />Filter.by_property(&quot;release_year&quot;).less_or_equal(year_range[1])<br />),<br />limit=SEARCH_LIMIT,<br />alpha=SEARCH_MODES[mode][1],<br />grouped_task=rag_prompt,<br />grouped_properties=[&quot;title&quot;, &quot;tagline&quot;],<br />)</pre><p>The result is a fully interactive recommendation app with no JavaScript experience required!</p><p>If you would like to learn more about Weaviate, check out the <a href="https://weaviate.io/developers/weaviate/quickstart">Weaviate Quickstart</a> and <a
