<figure><img alt="" src="https://cdn-images-1.medium.com/max/1024/1*LQj3YkCaJBmHeS2xhnAisQ.png" /></figure><h3>Introduction</h3><p>If you’ve ever worked with AI coding assistants like Cursor, GitHub Copilot, or Claude Code, you know they can be incredibly powerful, but also inconsistent. Sometimes it nails your code style, sometimes it forgets everything you’ve established.</p><p>This is where <a href="https://agents.md/">AGENTS.md</a> comes in. It’s an open standard for guiding AI coding agents. Think of it as a README for AI, a dedicated file that provides context, conventions, and instructions that help AI assistants work effectively on your coding projects.</p><p>In this article, I’ll show you how to use this AGENTS.md file to build Streamlit apps across all deployment targets: local development, Streamlit Community Cloud, and Streamlit in Snowflake (SiS).</p><h3>What is AGENTS.md?</h3><p>AGENTS.md is a simple, open format adopted by over 60,000 open-source projects, including repositories from OpenAI, Apache, and Google. It emerged from collaborative efforts across the AI software development ecosystem.</p><p>The key insight is that README.md files are for humans, while AGENTS.md is for AI agents. It contains the detailed context that AI coding assistants need: build commands, code conventions, testing patterns, and domain-specific knowledge.</p><h3>Where These Patterns Come From</h3><p>The Streamlit AGENTS.md isn’t theoretical. It’s built from real-world experience:</p><ul><li><a href="https://30daysofai.streamlit.app/"><strong>30 Days of AI</strong></a>: A 30-day learning series teaching AI app development with Streamlit and Snowflake Cortex (<a href="https://github.com/streamlit/30daysofai">GitHub</a>). Each day’s lesson contributed battle-tested patterns for chatbots, RAG systems, streaming responses, and agent orchestration.</li><li><strong>Experience in Streamlit Development</strong>: Patterns refined from building dozens of Streamlit applications over 2024–2025, including data dashboards, cheminformatics tools, ML apps, and enterprise applications deployed to Streamlit in Snowflake.</li></ul><p>Every pattern in this AGENTS.md has been validated in production. The AI_COMPLETE function, the universal connection pattern, the streaming chat interface: all emerged from solving real-world problems across different deployment environments.</p><h3>Why This Works</h3><p>The AGENTS.md approach works because:</p><ol><li><strong>Minimal friction</strong>: One file reference, a few questions, complete app</li><li><strong>Smart defaults</strong>: Caching, error handling, and UI components are inferred</li><li><strong>Environment-aware</strong>: Automatically handles SiS vs Community Cloud differences</li><li><strong>Complete output</strong>: Every app includes app.py, requirements.txt, README.md</li><li><strong>Consistent patterns</strong>: Same proven code patterns every time</li></ol><h3>Download AGENTS.md</h3><p>To proceed further, download the Streamlit AGENTS.md file from this <a href="https://github.com/dataprofessor/streamlit/blob/main/AGENTS.md">GitHub repo</a>:</p><p><a Universal Database Connection</h4><p>Works across all three deployment environments:</p><pre>@st.cache_resource<br />def get_session():<br />    try:<br />        from snowflake.snowpark.context import get_active_session<br />        return get_active_session()<br />    except:<br />        from snowflake.snowpark import Session<br />        return Session.builder.configs(<br />            st.secrets[&quot;connections&quot;][&quot;snowflake&quot;]<br />        ).create()</pre><h4>2. Snowflake Cortex LLM with AI_COMPLETE</h4><p>The recommended pattern for universal compatibility:</p><pre>import json<br />from snowflake.snowpark.functions import ai_complete<br /><br />@st.cache_data(show_spinner=False)<br />def call_llm(prompt: str, model: str = &quot;claude-3-5-sonnet&quot;) -&gt; str:<br />    df = session.range(1).select(<br />        ai_complete(model=model, prompt=prompt).alias(&quot;response&quot;)<br />    )<br />    response_raw = df.collect()[0][0]<br />    response_json = json.loads(response_raw)<br />    <br />    if isinstance(response_json, dict) and &quot;choices&quot; in response_json:<br />        return response_json[&quot;choices&quot;][0][&quot;messages&quot;]<br />    return str(response_json)</pre><h4>3. Chat Interface with Streaming</h4><pre>st.session_state.setdefault(&quot;messages&quot;, [])<br /><br />for msg in st.session_state.messages:<br />    with st.chat_message(msg[&quot;role&quot;]):<br />        st.markdown(msg[&quot;content&quot;])<br /><br />if prompt := st.chat_input(&quot;Your message&quot;):<br />    st.session_state.messages.append({&quot;role&quot;: &quot;user&quot;, &quot;content&quot;: prompt})<br />    with st.chat_message(&quot;user&quot;):<br />        st.markdown(prompt)<br />    <br />    with st.chat_message(&quot;assistant&quot;):<br />        response = st.write_stream(stream_response(prompt))<br />    <br />    st.session_state.messages.append({&quot;role&quot;: &quot;assistant&quot;, &quot;content&quot;: response})</pre><h4>4. SiS-Aware Code</h4><p>The AI automatically knows:</p><ul><li><strong>SiS deployment</strong> → No st.set_page_config()</li><li><strong>Community Cloud</strong> → Include st.set_page_config()</li><li><strong>Both</strong> → Use the universal connection pattern</li></ul><h3>Pattern Selection Guide</h3><p>Based on what you ask for, the AI selects the right patterns:</p><pre>+-------------------------+---------------------------------------------+<br />| You Say                 | AI Uses                                     |<br />+-------------------------+---------------------------------------------+<br />| chatbot, AI assistant   | Chat Interface + Streaming + AI_COMPLETE    |<br />+-------------------------+---------------------------------------------+<br />| dashboard,              | Basic App + Plotly/Altair Charts            |<br />| visualization           |                                             |<br />+-------------------------+---------------------------------------------+<br />| data analysis           | File Upload + DataFrame Styling             |<br />+-------------------------+---------------------------------------------+<br />| RAG, search documents   | Chat + Cortex Search + AI_COMPLETE          |<br />+-------------------------+---------------------------------------------+<br />| multipage               | st.navigation + Session State               |<br />+-------------------------+---------------------------------------------+<br />| Snowflake, SiS          | Omit st.set_page_config + Cortex patterns   |<br />+-------------------------+---------------------------------------------+</pre><p>Caching is added automatically wherever beneficial.</p><h3>Common Pitfalls Prevented</h3><p>The AGENTS.md teaches the AI to avoid common mistakes:</p><h4>1. Duplicate Widget Keys</h4><pre># AI learns to always add unique keys<br />st.text_input(&quot;Name:&quot;, key=&quot;first_name&quot;)<br />st.text_input(&quot;Name:&quot;, key=&quot;last_name&quot;)</pre><h4>2. Page Config in SiS</h4><pre># AI omits st.set_page_config() when targeting SiS<br /># (it's not supported)</pre><h4>3. Session State in Multipage Apps</h4><pre># AI initializes in main app.py, not individual pages<br />st.session_state.setdefault(&quot;df&quot;, None)<br />pg = st.navigation(...)<br />pg.run()</pre><h3>Conclusion</h3><p>Building a Streamlit app couldn’t be easier. With the AGENTS.md file, you can truly vibe code. Just describe what you want, answer a few questions, and watch your app come to life.</p><p>Instead of explaining patterns in detail, going through several rounds of promptings and hoping that AI gets it right.</p><p>Here’s what you get from using AGENTS.md:</p><ul><li><strong>Two modes</strong>: Quick instructions or guided questions</li><li><strong>Sequential flow</strong>: One question at a time, not overwhelming</li><li><strong>Smart inference</strong>: UI components and caching handled automatically</li><li><strong>Complete projects</strong>: App, requirements, and README ready to deploy</li><li><strong>Environment-aware</strong>: SiS compatibility built in</li></ul><p>Check out the examples/ folder in the <a href="https://github.com/dataprofessor/streamlit">repo</a> to see the chatbot and dashboard apps created using this exact flow.</p><h3>Resources</h3><ul><li><a href="https://agents.md">AGENTS.md Official Website</a></li><li><a href="https://docs.streamlit.io">Streamlit Documentation</a></li><li><a href="https://docs.snowflake.com/en/user-guide/snowflake-cortex/aisql">Snowflake Cortex AI</a></li><li><a href="https://docs.snowflake.com/en/developer-guide/streamlit/about-streamlit">Streamlit in Snowflake</a></li><li><a href="https://github.com/dataprofessor/streamlit/tree/main/examples">Example Apps</a>: Example apps built with AGENTS.md</li></ul><img alt="" height="1" src="https://medium.com/_/stat?event=post.clientViewed&amp;referrerSource=full_rss&amp;postId=04b7480f754e" width="1" /><hr /><p><a href="https://blog.streamlit.io/vibe-code-streamlit-apps-with-ai-using-agents-md-04b7480f754e">Vibe code Streamlit apps with AI using AGENTS.md</a> was originally published in <a href="https://blog.streamlit.io">Streamlit</a> on Medium, where people are continuing the conversation by highlighting and responding to this story.</p>
